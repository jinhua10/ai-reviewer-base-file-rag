## ⚠️ 需要注意的事项

### 1. 保留版权声明（必须）

在你的软件分发包中，**必须包含**：
- ✅ `NOTICE` 文件
- ✅ `LICENSE.txt` 文件（Apache 2.0 许可证）
- ✅ 或在软件的"关于"页面中显示版权信息

### 2. 不要使用项目商标

❌ **禁止：**
- "基于 Apache Lucene 官方版本"
- "Spring 认证产品"
- 使用第三方项目的 Logo

✅ **允许：**
- "使用 Apache Lucene 技术"
- "基于 Spring Boot 框架构建"
- 在文档中列出技术栈

### 3. 出口管制（仅适用于 BouncyCastle）

如果你的软件需要**出口到国外**：
- ⚠️ BouncyCastle 加密库可能受出口管制
- ✅ 中国境内使用：完全没问题
- ⚠️ 出口到美国、欧盟以外：可能需要咨询法律顾问

## 📝 推荐的合规实践

### 在你的产品中添加"关于"页面

```markdown
## 关于

本产品使用以下开源技术构建：

- Apache Lucene (Apache 2.0)
- Spring Boot (Apache 2.0)
- Apache Tika (Apache 2.0)
- 其他开源组件（详见 NOTICE 文件）

完整的开源许可证信息请查看软件安装目录下的 LICENSE 和 NOTICE 文件。
```

### 在软件安装包中包含

```
your-product/
├── bin/
├── config/
├── lib/
├── LICENSE.txt          ← 必须包含
├── NOTICE              ← 必须包含
└── licenses/           ← 推荐包含
    ├── Apache-2.0.txt
    └── MIT.txt
```

### 在网站/文档中说明

在产品网站的"许可证"或"法律信息"页面中：

```markdown
## 开源许可证

本产品采用以下开源组件：

| 组件 | 版本 | 许可证 |
|------|------|--------|
| Apache Lucene | 9.10.0 | Apache 2.0 |
| Spring Boot | 3.2.11 | Apache 2.0 |
| ... | ... | ... |

这些组件均采用商业友好的开源许可证，允许用于商业产品。
```
